# state

## state가 뭐냐 ? (정의)

- UI 화면을 결정하는 기준점이 되는 값

## js로 할 수 있는데 react의 state를 사용하는 이유는?(이유)

- js 는 직접 dom을 조작해야하지만 React는 state값만 바꾸면 UI를 자동으로 다시 계산해서 dom에 반영한다.
- 왜 그게 좋은가? 개발자는 그걸 지금 상태가 무엇인지만 신경쓰고 어떻게 바꿀지는 생각안해도된다.

## state가 동작하는 원리(구조)

1. state가 setState를 통해 변경된다.
2. REACT에게 상태 변경이 알려진다.
3. state가 선언된 컴포넌트가 다시 실행되면서 새로운 Virtual DOM을 생성한다.
4. 이전 Virtual DOM과 비교한다
5. 변경된 부분만 실제 DOM에 반영한다.
6. 브라우저는 DOM을 기반으로 화면을 그린다.

## state를 언제 쓰는지?(조건)

- 값이 변화하고 그 변화가 UI에 영향을 줄 때 사용한다.
- 반대로 UI에 영향을 주지 않거나 변화하지 않는 값은 state를 사용하지 않는다.

## state를 어디에 쓰냐? (실전)

- 내 프로젝트 piclog의 경우 posts는 변화될 수 있고 그에 따라 UI가 다르게 보여줄 수 있기 때문에 사용한다.
